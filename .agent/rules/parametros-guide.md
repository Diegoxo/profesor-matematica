# Requerimientos del sitio
- Dominio: profesordematematica.com.co
- El sitio web debe ser creado con HTML y CSS.
- El sitio web no debe tener JS, excepto para validaciones de los formularios que se requieran.
- El sitio web debe tener su favicon.
- El header y el footer debe ser el mismo para todas las paginas.
- No combinar css en html
- Los botones de WhatsApp y Llamar deben deter una clase unica para cada uno, que ningun otro elemento las tenga.
- Durante el desarrollo se deben dejar indicados los elementos donde iran las imagenes para luego ser reemplazadas manualmente.

# Requerimientos SEO
- El sitio web debe estar optimizado para SEO.
- Los enlaces internos deben conectarse entre sí para mejorar el SEO interno.
- Las URLs, deben ser limpias y amigables.
- El sitio web, debe ser apto para móviles.
- Se debe incluir un sitemap.xml actualizado.
- El sitio debe incluir un archivo robots.txt correctamente configurado.
- Las páginas internas deben incluir breadcrumbs hechos con HTML.
- El sitio web debe tener el archivo .htaccess, para las URLs limpias.
- Los tiempos de carga deben ser lo más rápido posibles.
- Todas las páginas deben tener un título único y una meta descripción optimizada para SEO.
- Todas las paginas deben tener una meta descripción orientada a SEO.
- El sitio web debe tener implementado rich snippets todas las paginas que sea importante.

# Requerimientos de UI
- El sitio debe utilizar una paleta basada en azules.
- El sitio web debe sentirse muy reactivo, muy vivo, que las cosas se sientan vivas.
- El sitio debe sentirse moderno, minimalista y elegante.
- Los botones deben tener animaciones suaves al pasar el cursor encima.
- La barra de navegacion no debe estar en un menu hamburguesa
- El boton de whatsapp y del numero de telefono deben estar fijos en la parte de abajo, uno seguido del otro y ocupar el ancho de la pantalla en la version movil. 
- Los botones de llamar y whatsapp deben tener un efecto de brillo que se mueve al lo largo de los botones.

# Requerimientos de UX
- El sitio web debe ser Mobile first
- El header en la versión móvil, no puede tener textos que se solapen o que se vean mal.
- Los CTA deben estar visibles y repetidos estratégicamente.
- El header de las páginas debe quedarse fijo, no importa si se baja la página.
- Al tocar el logo de WhatsApp o darle clic debe redirigir a una conversación en WhatsApp.
- El contenido debe transmitir confianza y experiencia.
